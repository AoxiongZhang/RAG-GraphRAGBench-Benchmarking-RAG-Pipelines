# RAG Test Benchmark

This repository contains experimental code and evaluation pipelines for a retrieval-augmented generation (RAG) system.

The project uses RAGAS to conduct comparative evaluations of different RAG components and techniques.
In addition, it includes the construction and evaluation of knowledge graphs, as well as corresponding GraphRAG experiments.

The repository is intended as a research and benchmarking framework for analyzing RAG behavior under different configurations, datasets, and evaluation settings.

## Step 2: Data Preparation

The datasets used in this study are located in the `data/` directory.

### 2.1 Synthetic QA Dataset Generation

To simulate realistic RAG evaluation scenarios, a synthetic question–answer test dataset is generated by running `testdata_generation.ipynb`.

This script generates a QA dataset consisting of:
- the first 20 single-hop questions, and
- the last 20 multi-hop questions.

The generated dataset is saved to:

`evaluate_dataset/test_dataset.csv`

---

### 2.2 Dataset Cleaning and Correction

Due to grammatical errors and noisy user inputs, the generated dataset is manually filtered and corrected.

The cleaned and corrected version of the dataset is saved as:

`evaluate_dataset/test_dataset_cor.csv`

This corrected dataset is used for comparison against the noisy dataset in subsequent experiments.

---

### 2.3 External Benchmark Dataset (TechQA)

An additional benchmark-style dataset is generated from a Hugging Face dataset by running `testdata_generation_TechQA.ipynb`

The resulting dataset is saved to:

`evaluate_dataset/test_dataset_ragbench.csv`

This dataset is used to further evaluate the generalization of the RAG pipeline.

## Step 3: RAG Testing
During the evaluation stage, the corresponding `*_test.ipynb` notebooks are executed sequentially.

Each notebook evaluates a specific component of the RAG pipeline, and all evaluation results are saved to the `evaluate_results/` directory.

## Step 4: GraphRAG Testing

### Configuration

GraphRAG experiments require a running Neo4j instance.

Before running GraphRAG-related notebooks or scripts, configure the following environment variables in the `.env` file:

- `NEO4J_URI`
- `NEO4J_USERNAME`
- `NEO4J_PASSWORD`

These variables are used to establish a connection to the Neo4j database for knowledge graph construction and querying.

### Rule-base and IE

To construct the knowledge graph using a rule-based approach, run the `KG_Construction/Rule_based.ipynb` notebook.

After execution, follow the final steps in the notebook to manually create the graph in Neo4j.

Run the `Rule_based_KG_test.ipynb` notebook to obtain the evaluation results.

### LLM-based
To construct the knowledge graph using a LLM-based approach, run the `KG_Construction/LLM_based.ipynb` notebook.

After execution, follow the final steps in the notebook to manually create the graph in Neo4j.

Run the `LLM_based_KG_test.ipynb` notebook to obtain the evaluation results.
